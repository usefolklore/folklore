## P1 — web-gating (why this isn't folklore's axis)

Fair metric = fallback-rate at matched false-accept (swept threshold); raw fallback alone is rigged (reckless caching wins it). On the 32-event toy stream the cosine cache and **LangChain measure identically** (LangChain == the cosine proxy exactly — proxy validated). folklore reads 1.0 — but that is a **metric/regime mismatch, not a defeat**: its retrieval `satisfaction` is calibrated for a populated graph + the deny pipeline, is **not cosine-comparable**, and is too compressed to sweep as a 0–1 threshold (a *correct* top hit scores ~0.42). Forcing folklore onto the cosine-cache axis mismeasures it; reporting the 1.0 as a loss would inflate the competitors via a rigged regime. Single-user web-gating on a synthetic micro-graph is simply not a valid head-to-head — folklore's gating runs on a large populated graph in production. The decisive, *valid* comparisons are P2 (provenance) and P3 (federation).

**mem0 (now runnable on py3.13) is MEASURED but NONDETERMINISTIC.** Every mem0 write goes through an LLM (Ollama qwen2.5:7b) that extracts/dedups facts. A 4-seed variance pass: cosine and LangChain are perfectly stable (fallback@FA≤0.05 = 0.4688, min==max); mem0 ranges 0.906–1.0 with correct-serve just 0–9% — both nondeterministic AND worse here. Balanced read (not 'mem0 is bad'): (1) LLM-mediated writes → nondeterministic + costly per write; (2) mem0 targets *conversational* memory, not Q→A paraphrase-cache recall, so this micro-harness underserves it. Honest claim: mem0 is a single-user, LLM-mediated, nondeterministic store with no provenance/federation — the deterministic similarity-cache column is owned by LangChain==cosine. No cherry-picking, either way.

## PD — latency + cost

**Latency** is parity at the retrieval core (all MiniLM-bound). folklore's CLI number includes Node boot and is NOT its production path (MCP/IPC); its retrieval core is the documented 11 ms median. Compare cores, not transports.

**Write cost (structural):** folklore / LangChain / cosine = a free local embed per write; **mem0 / Letta run an LLM extraction per write** — a real token/GPU cost borne only by the LLM-memory tools, invisible in retrieval latency.

**Paid trips saved:** fallback ~0.47 @FA<=0.05 -> ~53% of repeat/paraphrase trips saved; hit 0.97 -> ~97% of trips saved vs web-every-time; silo tools flat ~0.31.
