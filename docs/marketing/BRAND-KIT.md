# Akashik — brand kit

The visual contract every shipped surface adheres to. Generated by
applying taste-skill to the SITE-REDESIGN-SPEC.md, with constants
preserved from the current `docs/index.html` where they were already
correct.

**Configuration (per taste-skill dials):**

- `DESIGN_VARIANCE`: **8** — asymmetric layouts default; centered hero/H1 banned
- `MOTION_INTENSITY`: **6** — fluid CSS + scoped Framer-Motion-grade choreography on isolated leaves
- `VISUAL_DENSITY`: **4** — generous breathing room; no Cockpit packing

**Status:** brand kit finalized; supersedes ad-hoc decisions in
SITE-REDESIGN-SPEC.md where they conflict.

---

## 1 · Brand pillars (the non-negotiables)

The five attributes every design decision must defend:

| # | Pillar | Practical implication |
|---|--------|------------------------|
| 1 | **Premium-OSS, not corporate-SaaS** | Bun, Linear, Astro, Excalidraw — not Salesforce, HubSpot, Stripe-marketing. Confidence through restraint. |
| 2 | **Graph-native** | The product is a graph. Lines, nodes, edges, density gradients should appear as design language — never as decoration. |
| 3 | **Cooperative, not crypto** | Lineage is Napster / eMule / BitTorrent. No purple-blue, no neon, no "decentralized" dressing. |
| 4 | **Data-truthful** | Numbers shown are real (live counters, live latency, live bench). Never demo placeholder data on production surfaces. |
| 5 | **Quiet motion** | Movement signals state changes (peer joined, query landed, edge activated). Movement that is decoration is banned. |

## 2 · Color tokens

Single accent family. Saturation capped at 70%. No purple/blue under
any condition. The "Lila ban" applies hard.

### 2.1 Tokens (lock these — do not introduce new colors)

```css
:root {
  /* Ink scale (dark surfaces) */
  --ink-900:        #0c0c14;   /* base — NEVER pure black */
  --ink-800:        #14141f;   /* card / elev-2 */
  --ink-700:        #1c1c28;   /* elev-3 */
  --ink-500:        #3a3a4a;   /* hairline / divider */
  --ink-mute:       #a4a4af;   /* secondary text on dark — bumped from #8c8c98 for AA */

  /* Paper scale (light surfaces) */
  --paper:          #f7f4ec;   /* warm off-white */
  --paper-elev:     #ffffff;
  --paper-ink:      #161619;   /* primary text on paper */
  --paper-mute:     #5b5b63;   /* secondary text on paper */

  /* Accent — single hue, single family */
  --accent:         #34d399;   /* emerald, S~69%, L~52% */
  --accent-ink:     #0f6a46;   /* AA-text on paper backgrounds */
  --accent-warm:    #d9a255;   /* amber — DATA ANNOTATIONS ONLY (bench, charts, callouts) */
  --accent-dim:     rgba(52, 211, 153, 0.14);
  --accent-emit:    rgba(52, 211, 153, 0.55);  /* graph node glow ceiling */

  /* Borders */
  --border-dark:    rgba(255, 255, 255, 0.08);
  --border-light:   rgba(22, 22, 25, 0.09);
}
```

### 2.2 Banned colors

- `#000000` (use `--ink-900`)
- Any purple, indigo, violet hex — **no exceptions**
- Neon green / lime greens (saturation > 80%)
- Default Tailwind `bg-blue-500` and friends — they're the AI tell
- Gradient fills on text (banned per taste-skill §7)
- Multi-stop background gradients with > 2 colors

### 2.3 Allowed amber usage

`--accent-warm` is for **data annotations only**:

- Benchmark bars showing comparison cohorts
- "Stale" badges on graph hits older than the room's freshness window
- Highlight ring on hovered chart datapoints

It is NOT a second brand color. It NEVER appears in CTAs, body copy,
section backgrounds, or icons.

## 3 · Typography

### 3.1 Stack

```css
--display: 'Outfit', -apple-system, BlinkMacSystemFont, 'SF Pro Display', system-ui, sans-serif;
--mono:    'JetBrains Mono', 'SF Mono', ui-monospace, monospace;
```

Outfit is on the taste-skill allow-list. Inter is **banned** per §7.
Serif fonts are banned (this is a technical UI). System fallbacks
mean a font-load failure still ships premium-feeling typography.

### 3.2 Scale

Use `clamp()` for fluid scaling. Lock to these tokens — do not
introduce intermediate sizes.

| Token | Mobile (375 px) | Desktop (1440 px) | Use |
|-------|-----------------|-------------------|-----|
| `--t-display`   | `2.4rem` | `5.6rem` | Hero H1 only — single instance per page |
| `--t-section`   | `1.8rem` | `3.2rem` | Section H2 |
| `--t-sub`       | `1.3rem` | `2rem`   | Sub-section H3 |
| `--t-claim`     | `1.1rem` | `1.25rem`| Hero claim line (accent emerald) |
| `--t-body`      | `0.95rem`| `1.05rem`| Body copy |
| `--t-meta`      | `0.85rem`| `0.92rem`| Captions, counters, tag pills |
| `--t-micro`     | `0.72rem`| `0.78rem`| Eyebrows, badges, tag uppercase |

```css
:root {
  --t-display:  clamp(2.4rem,  6vw, 5.6rem);
  --t-section:  clamp(1.8rem,  3.5vw, 3.2rem);
  --t-sub:      clamp(1.3rem,  2.2vw, 2rem);
  --t-claim:    clamp(1.1rem,  1.4vw, 1.25rem);
  --t-body:     clamp(0.95rem, 1.1vw, 1.05rem);
  --t-meta:     clamp(0.85rem, 0.95vw, 0.92rem);
  --t-micro:    clamp(0.72rem, 0.8vw, 0.78rem);
}
```

### 3.3 Tracking + leading

| Class | letter-spacing | line-height |
|-------|----------------|-------------|
| `.t-display` | `-0.035em` | `0.98` |
| `.t-section` | `-0.022em` | `1.05` |
| `.t-sub`     | `-0.018em` | `1.15` |
| `.t-body`    | `0`        | `1.55` |
| `.t-mono`    | `0`        | `1.5`  |
| `.t-meta`    | `0.005em`  | `1.4`  |
| `.t-micro`   | `0.14em`   | `1.2`  (uppercase) |

The H1 is sized large (5.6rem desktop) but **muted via weight** —
600, not 800. Per taste-skill §7: "The first heading should not
scream. Control hierarchy with weight and color, not just massive
scale."

### 3.4 Body copy width

Body paragraphs MUST respect `max-width: 65ch` for readability.
Marketing landing pages pad more — `max-width: 56ch` for hero subs
and section intros to keep the eye-line snug.

## 4 · Spacing & layout

### 4.1 Section rhythm

| Region | Vertical padding |
|--------|------------------|
| Hero   | `clamp(3rem, 7vw, 5rem)` top/bottom (within `min-h-[100dvh]`) |
| Major sections | `clamp(4rem, 10vw, 8rem)` top/bottom |
| Sub-sections inside a major | `clamp(2rem, 4vw, 3rem)` |

### 4.2 Container

```css
.shell {
  max-width: 1280px;       /* --max-w */
  margin-inline: auto;
  padding-inline: clamp(1.25rem, 4vw, 3rem);
}
```

**No `h-screen`. Anywhere.** Hero uses `min-h-[100dvh]` per
taste-skill §10. iOS Safari layout-jumping is a known foot-gun.

### 4.3 Grid bias (asymmetric per VARIANCE 8)

Hero: `grid-template-columns: 1.4fr 1fr` (NOT 1fr 1fr).

Architecture / Live Network sections: `1fr 1fr` is acceptable when
the two halves carry equal-weight content (graph + query box). For
all other 2-up sections, prefer asymmetric (`1.3fr 1fr`, `2fr 1fr`,
`1fr 1.6fr`).

**3-column equal-card rows are BANNED** per taste-skill §7. For the
"how it works in 3 steps" section, use a vertical stack with offset
left margins (each step indented further), or a 2-column zig-zag
layout with the third step centered below.

### 4.4 Mobile collapse

Every asymmetric layout above `md:` collapses to **strict
single-column** below 900 px. No partial responsive layouts. The
`order` property handles DOM-vs-visual reordering (gif renders below
text on mobile, etc. — see SITE-REDESIGN-SPEC.md §6.4).

## 5 · Motion language

`MOTION_INTENSITY: 6` — fluid CSS for most surfaces, scoped advanced
choreography for the live demo. No window-scroll listeners ever.

### 5.1 Easing tokens

```css
:root {
  --ease:       cubic-bezier(0.16, 1, 0.3, 1);   /* default, "out-quint" */
  --ease-bump:  cubic-bezier(0.34, 1.56, 0.64, 1); /* overshoot, for status badges */
  --ease-linear: linear;                          /* monotonic loops only (rotate, marquee) */
}
```

### 5.2 Duration scale

| Token | Duration | Use |
|-------|----------|-----|
| `--d-fast`   | 180 ms | Hover, focus, button-state |
| `--d-base`   | 320 ms | Reveal, fade-in, layout shift |
| `--d-slow`   | 600 ms | Section reveal, cascade entry |
| `--d-pulse`  | 800 ms | Counter tick, status badge pop |

### 5.3 Hardware-accelerated only

Animate **transform + opacity only**. Never width/height/top/left.
The current `pulse-ring` keyframe on `.brand-mark::after` is
compliant (uses `transform: scale()` + `opacity`).

### 5.4 Scroll-triggered reveal (already in place — keep)

The page uses `data-reveal` + `--d` CSS variable + IntersectionObserver
for staggered entry. Stagger timing:

```css
[data-reveal] {
  opacity: 0;
  transform: translateY(12px);
  transition: opacity var(--d-slow) var(--ease) calc(var(--d, 0) * 80ms),
              transform var(--d-slow) var(--ease) calc(var(--d, 0) * 80ms);
}
[data-reveal].is-revealed {
  opacity: 1;
  transform: translateY(0);
}
```

`--d` is the stagger index (0, 1, 2, 3…); each step adds 80 ms.

### 5.5 Perpetual motion (use sparingly)

Per taste-skill §4, infinite micro-loops belong on:

- Brand mark pulse-ring (already in place — keep)
- Live counter on tick (counter-pulse keyframe per
  SITE-REDESIGN-SPEC.md §2.6)
- §11 live graph on idle (auto-rotate at 0.6 deg/sec — pauses on
  hover)
- Live status indicator dots (1.5 s pulse cycle)

Each must be **isolated in its own Client Component** (when this
moves to React/Next), wrapped in `React.memo`, and never trigger
parent re-renders.

### 5.6 Forbidden motion

- Custom mouse cursors (taste-skill §7) — banned
- Auto-playing video backgrounds — banned
- Parallax background images (the `MOTION_INTENSITY 8-10` tier) —
  not at this dial setting
- Infinite glow / breathing on every element — banned (the
  AI-design tell)

## 6 · Component grammar

### 6.1 Button hierarchy

Three button shapes. No others.

```
btn-primary    — emerald fill, ink-900 text, semibold 600
btn-ghost      — transparent, 1px border, ink-mute text → ink on hover
btn-quiet      — text-only, ink-mute, no border, accent on hover
```

```css
.btn {
  display: inline-flex;
  align-items: center;
  gap: 0.5rem;
  padding: 0.75rem 1.4rem;
  font-family: var(--display);
  font-weight: 600;
  font-size: 1rem;
  border-radius: 999px;        /* full pill */
  border: 1px solid transparent;
  transition: transform var(--d-fast) var(--ease),
              background var(--d-fast) var(--ease),
              border-color var(--d-fast) var(--ease);
  cursor: pointer;
}
.btn:active   { transform: translateY(1px) scale(0.98); }
.btn:focus-visible { outline: 2px solid var(--accent); outline-offset: 3px; }

.btn-primary {
  background: var(--accent);
  color: var(--ink-900);
}
.btn-primary:hover {
  background: color-mix(in oklab, var(--accent) 90%, white);
}

.btn-ghost {
  background: transparent;
  border-color: var(--border-dark);
  color: #e8e8ed;
}
.btn-ghost:hover {
  border-color: var(--accent);
  color: var(--accent);
}
```

Tactile feedback on `:active` is **mandatory** per taste-skill
Rule 5: `translateY(1px) scale(0.98)`.

### 6.2 Counter (live signal pattern)

See SITE-REDESIGN-SPEC.md §2.6 for full implementation. Visually:

- Mono font, `--t-meta` size, `--ink-mute` color
- Leading `·` glyph (separator, not bullet)
- `:hover` → `#e8e8ed` color + underline
- Tick state → `counter-pulse` keyframe (`-2px translateY`, accent
  color flash, 800 ms)
- Loading state → mono `—` placeholder (no skeleton, no spinner)

### 6.3 Install strip (segmented pill)

See SITE-REDESIGN-SPEC.md §2.5. Visually anchored to the same
button-radius vocabulary (`6px` outer, `4px` inner segments).

### 6.4 Cards (USE SPARINGLY)

Per taste-skill Rule 4: cards only when elevation hierarchy demands
it. Akashik surfaces using cards:

- Result cards in §11 query results (`.result-card`)
- Chapter cards (existing) — review per Rule 4 before keeping
- Comparison-table rows — NOT cards; `divide-y` instead

When a card IS used:

```css
.card {
  background: var(--ink-800);
  border: 1px solid var(--border-dark);
  border-radius: 16px;
  padding: 1.25rem 1.5rem;
  /* Diffusion shadow, NOT generic drop-shadow */
  box-shadow: 0 20px 40px -15px rgba(0, 0, 0, 0.45);
}
```

For light sections: `--paper-elev` background, `--border-light`
border, `box-shadow: 0 20px 40px -15px rgba(22, 22, 25, 0.08)`.

### 6.5 Anti-card sections

Per the redesign, the comparison table and benchmark bars use
`border-t` + `divide-y` — no card containers. Numbers breathe
without being boxed.

### 6.6 Code blocks (monospace surfaces)

```css
.code {
  background: var(--ink-800);
  border: 1px solid var(--border-dark);
  border-radius: 8px;
  padding: 1rem 1.25rem;
  font-family: var(--mono);
  font-size: 0.92rem;
  line-height: 1.6;
  color: #e8e8ed;
  overflow-x: auto;
}
```

The leading `$` for shell commands is a **design choice**, not
copy-pasteable. JS prefixes the text without `$` when copying to
clipboard.

### 6.7 Form inputs

Per taste-skill Rule 6: label above input, helper below, error
below. For Akashik surfaces:

```html
<label for="ask-input">Ask the network</label>
<input id="ask-input" type="text" />
<small class="helper">Try: "LSTM quench detection"</small>
<small class="error" hidden>Couldn't reach the network — try again?</small>
```

```css
.ask-input {
  width: 100%;
  padding: 0.9rem 1.2rem;
  background: var(--ink-800);
  border: 1px solid var(--border-dark);
  border-radius: 12px;
  color: #e8e8ed;
  font-family: var(--display);
  font-size: 1rem;
  transition: border-color var(--d-fast) var(--ease);
}
.ask-input:focus {
  outline: none;
  border-color: var(--accent);
  box-shadow: 0 0 0 3px var(--accent-dim);
}
```

## 7 · Iconography (no emoji rule)

**ANTI-EMOJI POLICY [CRITICAL].** Per taste-skill §2.

The current SITE-REDESIGN-SPEC.md uses these glyphs in copy:

| Glyph | Where | Replacement |
|-------|-------|-------------|
| `☆`   | "Star on GitHub" CTA | inline SVG of a star icon (Phosphor `Star` filled outline, `strokeWidth: 1.6`) |
| `→`   | CTA arrow | inline SVG (already present in current page) |
| `⎘`   | Copy-button glyph in install strip | Phosphor `Copy` icon (16×16, `strokeWidth: 1.5`) |

**Inline SVG primitives** for the 6–8 icons the page actually needs.

zero-dep landing page can't justify. Inline SVG with consistent
`stroke-width: 1.6` and 16×16 viewBox is the right answer.

- `width="14" height="14"` for inline-text icons (CTAs, captions)

- `stroke-width="1.6"` always (NOT 1.5 — emerald accent reads better at 1.6)

## 8 · Imagery & illustration

### 8.1 Hierarchy of media

In order of preference for any given content slot:

1. **Live data visualization** (graph, counter, latency) — strongest signal
2. **Recorded GIF demo** (scene-claude.gif, scene-prompt-hook.gif) — strong proof
3. **Inline SVG diagram** (architecture, flow, lineage) — clean, brandable
4. **Static illustration** (hand-crafted SVG, light-touch) — used sparingly for thesis sections
5. **Photography** — banned on the marketing site (does not match the brand)

### 8.2 GIF treatment

```css
.demo-gif {
  border-radius: 12px;
  box-shadow: 0 4px 24px rgba(0, 0, 0, 0.45);
  /* IMPORTANT: explicit width/height to prevent CLS */
}
```

GIFs MUST:

- Live in `/demo/` (relative path, NEVER `raw.githubusercontent.com`)
- Have explicit `width` + `height` attributes
- Have `loading="lazy"` for below-the-fold, `loading="eager"` for hero
- Be wrapped in `<figure>` + `<figcaption>` for semantic correctness
- Keep duration ≤ 30 s (WCAG 2.2.2 — over 30 s requires controls)

### 8.3 Static-image placeholders

Per taste-skill §7: NO Unsplash. If a placeholder is needed during
development, use:

```
https://picsum.photos/seed/{descriptive-slug}/800/600
```

For production: **no stock photos**. If an image is needed, generate
it (see §9.5 image-gen brief) or commission it.

### 8.4 SVG diagrams (the brand voice for illustrations)

When inline SVG diagrams are needed (architecture flow, network
topology, install path), use **single-color line art** in the
emerald accent on the appropriate ink/paper background:

- Stroke: `var(--accent)` at 0.55 opacity for soft edges, 1.0
  opacity for emphasis
- Stroke-width: `1.3` for delicate diagrams, `1.6` for hero-scale
- No fills (or `--ink-900` cutouts to suggest depth)
- No gradients (banned on text per §7; banned on diagrams as the
  brand convention)

The favicon (`docs/index.html` line 10) is the canonical example —
6-peer mesh in single-color emerald lines on `--ink-900`. Replicate
this voice for any future diagrams.

## 10 · Voice & copy patterns

### 10.1 Voice attributes

| Be | Don't be |
|----|----------|
| Concrete (numbers, dates, citations) | Aspirational ("imagine a world where…") |
| Confident (declarative sentences) | Hedging ("we believe", "we think") |
| Technical (use the real terms) | Soft-explainer ("kind of like a database") |
| Cooperative (we, the network) | Singular-hero ("we built this for you") |
| Wry on cost ("zero tokens", "no one pays twice") | Earnest on cost ("we save you money") |

### 10.2 Locked headlines + claims

These strings are load-bearing across README / index.html / OG card
/ social-launch pack. Change one place, change everywhere.

- **H1:** *The globally accumulating knowledge network. For AI agents — and humans.*
- **Wedge claim:** *Deep research in 11 ms. Cited. Zero tokens.*

### 10.3 Banned words / phrases

Per taste-skill §7 + positioning-draft.md anti-patterns:

- **Filler:** *Elevate, Seamless, Unleash, Next-Gen, Empower, Leverage, AI-powered, Future of, Revolutionary, Disruptive*
- **Category:** *Decentralized AI, Web3, Blockchain (when not technically accurate)*
- **Soft:** *Kind of, Sort of, Basically, Essentially, Just, Really, Very*
- **Generic:** *Acme, Nexus, SmartFlow, John Doe, Sarah Chan, Jack Su*
- **Self-praise:** *Best-in-class, World-class, Industry-leading*

### 10.4 Numbers in copy

Always concrete, never round-fake:

- ✓ `11 ms p50`, `75.22% NDCG@10`, `4,891 chunks`, `47.2% NDCG@10`
- ✗ `99.99%`, `~50%`, `1234567`, `1,000,000+ users`

When the data isn't yet available (cold-start), say so. **"Be the
first peer"** is a stronger empty state than `0 peers online`.
