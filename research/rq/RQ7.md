## RQ7 — Associative memory, retrieval-augmentation, and what a federated inference cache inherits from them

**Answer.** The literature firmly supports *one* of Folklore's structural bets, is *directionally suggestive but untested* on a second, and leaves the third — the load-bearing one — open. (1) "The reranker is already one modern-Hopfield step" is a restatement of a published theorem: Ramsauer et al. prove the continuous modern-Hopfield update *is* transformer softmax-attention, so this is a known equivalence, not a novel framing (arXiv 2008.02217). This is the only bet the cited literature *proves*. (2) The hypothesis that models ingesting retrieved context as first-class evidence out-*calibrate* models that cannot is only weakly/indirectly supported: RETRO, In-Context RALM, and Memorizing Transformers all show retrieval improves *language modeling* (perplexity/loss), and In-Context RALM notes retrieval gives a natural source-*attribution* mechanism — but none of the four papers measures *calibration*, and neither perplexity gains nor attribution entails calibration gains. So the calibration bet is suggestive, not supported. (3) The **distributed Hopfield capacity theorem** Folklore records as open is genuinely open: no source gives a capacity bound for a sharded/federated associative memory. In fact none gives a capacity *theorem* for a kNN retrieval cache at all — only Ramsauer gives one, and only for a single-node Hopfield associative memory.

**What the literature shows.** Ramsauer et al. (arXiv 2008.02217, "Hopfield Networks is All You Need", Hochreiter group, JKU Linz) is the theoretical anchor: the continuous modern-Hopfield net stores exponentially many patterns in the dimension of the associative space, retrieves a pattern in one update, and has exponentially small retrieval errors, with an update rule provably equivalent to transformer attention and a three-way taxonomy of fixed points (a global fixed point averaging all patterns, metastable states averaging a subset, and fixed points storing a single pattern). This is what *confirms* the "reranker = one Hopfield step" observation outright.

The three retrieval papers span invasive to non-invasive. RETRO (arXiv 2112.04426, Borgeaud et al., DeepMind) is most invasive: a frozen BERT retriever feeds chunk-level neighbours from a 2-trillion-token database through a chunked cross-attention mechanism, reaching performance comparable to GPT-3 and Jurassic-1 on the Pile with 25× fewer parameters. Memorizing Transformers (arXiv 2203.08913, Wu, Rabe, Hutchins, Szegedy, ICLR 2022) is intermediate: an approximate kNN lookup into a non-differentiable external memory of recent (key,value) pairs, with gains improving steadily up to 262K tokens of memory across C4, arXiv math, PG-19 books, GitHub code, and Isabelle theorems — and the model uses newly defined functions/theorems at test time without retraining. In-Context RALM (arXiv 2302.00083, Ram et al., AI21, TACL) is least invasive and most Folklore-shaped: it leaves the LM architecture unchanged, prepends documents retrieved by a frozen off-the-shelf retriever, reports "surprisingly large" gains across model sizes and corpora, and frames retrieval as a natural source-attribution mechanism. (Caveat: my fetch of 2302.00083 returned only abstract-level text — no perplexity numbers, retrieval-stride values, or reranker details — so I cite no specifics there.)

The sources agree that external retrieval substitutes for parameters/memorization; they differ on architectural cost — RETRO trains cross-attention, Memorizing Transformers adds a kNN-attention layer, In-Context RALM needs nothing but the context window. Folklore sits at the In-Context RALM end: it injects provenance-tagged graph hits into the prompt, no model surgery. Note the cost spectrum is about *language-modeling* improvement; none of the three reports a *calibration* metric.

**How it maps to Folklore.** CONFIRMS: the Hopfield equivalence is real and published (so the reranker observation is correct but not novel). PPR and EigenTrust are genuinely fixed-point iterations (both converge by power iteration to a stationary/principal eigenvector), so calling them, with cache eviction, "energy-descent fixed points" is a reasonable *analogy* — though "fixed-point" is the literal part and "energy-descent" (a Lyapunov/gradient reading) is the looser one. EXTENDS: Folklore generalises the single-node retrieval cache (Memorizing Transformers' 262K memory; In-Context RALM's prepend) into a *federated* one whose memory is a CRDT-synced graph across peers — a regime none of these papers test. CHALLENGES: the only capacity/error *guarantee* in these sources is Ramsauer's, and it is per-node, for a Hopfield associative memory over a fixed pattern set — not for a kNN retrieval cache, and not for a federated one. RETRO's 2T tokens and MT's 262K are empirical database/memory sizes, not capacity theorems. Folklore's memory is adversarial, churning, and trust-weighted (EigenTrust), so the clean exponential-capacity / one-step / exponentially-small-error guarantees do not transfer for free, and the single-node theorem does not entail a distributed one.

**Open questions / the falsifier.** The sharpest open question is Folklore's own, sharpened: *as N peers contribute, does effective retrieval capacity grow, plateau, or collapse?* Ramsauer's bound scales capacity with the dimension of one associative space and says nothing about whether unioning M peers' shares yields M× capacity or interference-driven metastable collapse — retrieval sliding toward the global-averaging fixed point (a blurred mean) instead of a specific pattern. That collapse is the named failure: if adding peers pushes retrieval toward metastable subset-averages, shared inference degrades to mush precisely when the network is largest.

The falsifier: hold query and local graph fixed, vary only federated peer count N, and measure retrieval *quality* — recall_any@k (a real metric, `src/domain/eval-metrics.ts`) and reranker satisfaction. If quality monotonically declines as N grows past a threshold (interference dominating), the "more peers = more usable memory" thesis is false and Folklore needs an explicit capacity/partitioning regime, not an open TODO. Status (corrected): a peer-count sweep N∈[1..64] *already exists* (`bench/bench-memtool-federation.py`), but it is self-labelled SIMULATOR — peer behaviour and CRDT sharing are *modelled* and only the MiniLM-cosine recall decision is *measured* — and it scores cooperative cache *hit-rate*, which is monotone-up by construction (it reports ~90% at 64 peers vs ~18% alone) and therefore *cannot* exhibit the capacity-collapse mode. So the capacity falsifier above — real federated retrieval quality vs N — has **not** been run; the existing simulated hit-rate sweep is not a test of it, and importing its numbers as evidence against collapse would conflate simulator with measurement. The corollary calibration test: ablate only the provenance tags (provenance-tagged vs identical untagged context) and measure calibration; if tagging does not improve calibration, the first-class-evidence hypothesis fails. This one is genuinely unrun, and no cited paper provides a calibration baseline for it. Both are runnable on the existing harness; neither has been run as a *measured* (non-simulated) capacity/calibration experiment.
