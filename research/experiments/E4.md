# E4 — MEASURED (not simulated) reuse savings net of verification cost

## Objective & hypothesis

**Objective.** Measure the *net* end-to-end saving of cooperative (federated) reuse
versus an isolated, well-tuned local cache, with **both arms held to the same 1%
error guarantee**, and with retrieval **+ verification** overhead subtracted from
the regeneration cost. This converts the in-repo simulator figures
(+17.5 pts correct-resolve, +5.44M tokens "reused", 1.0% false-admit from
`bench/bench-compounding-real.mjs` / `-graded.mjs`) into a wall-clock / token
saving, which RQ4 (`research/rq/RQ4.md`) explicitly flags is *not* what those
counts are.

**Falsifiable hypothesis (H1).** At a matched 1% false-accept budget, cooperative
net saving per resolved query exceeds the isolated net saving by ≥ 20% relative,
*after* retrieval and verification cost are priced in. **Falsified if** (a) once
verification is priced in, cooperative net saving ≤ isolated (overhead erases the
gap), **or** (b) holding false-accept ≤ 1% forces the federated operating-point
recall down to the isolated recall (the federation premium is a threshold
artifact).

## What this reuses vs what to build

**Reused as-is.**
- `bench/bench-vcache-compare.mjs` — the *matched-error* backbone. Runs COLD vs
  SINGLE-NODE (vCache-like, deposits to self) vs FEDERATED (CRDT, deposits to all
  peers) on the **identical** paraphrased BEIR+qrels stream (`SIGMA=0.033`,
  `para()` = always-paraphrase), sweeps `GRID` (0.95→0.30), returns `opPoint` =
  max recall with `false_accept ≤ BUDGET`. `--budget 0.01` gives the 1% guarantee.
- `bench/bench-memtool-latency.py` — **measured** per-op costs: `D1` retrieval
  latency (cosine/LangChain in-proc; folklore CLI end-to-end labeled "includes
  Node boot, NOT production; core 11 ms median"), `D2` write cost, `D3`
  trips-saved = `1 − fallback`.
- `bench/bench-compounding-real.mjs` — its token model (`C_RESEARCH=8000`,
  `C_RECALL=200`, `compute_saved_tok = correct·(C_RESEARCH−C_RECALL)`) is the
  structure to extend; those constants are **assumed**, not measured — E4 replaces
  them.
- `src/domain/cross-rerank.ts` `rerankMatches(query, matches, docTextOf, scorer)` —
  the cross-encoder pass is the concrete **verifier** whose cost we meter.
- `src/domain/energy-gate.ts` `energyGate` — reference admission rule for ablation.

**To build (new).** Add `bench/bench-net-savings.mjs` (slots beside
`bench-vcache-compare.mjs`, reusing its loader, `para()`, `drawQ()`, `runCurve`).
It (1) re-runs `runCurve(false)` / `runCurve(true)`, picks each arm's 1%-budget
operating point, then (2) computes a **net-savings ledger** per query using
measured unit costs (below). Add one measured-cost probe,
`bench/probe-unit-costs.mjs`, that times: a regeneration call (`C_regen`, tokens +
ms), a retrieval call (`C_retr`), and a verification rerank over top-K
(`C_verify`, via `rerankMatches` with a real `CrossEncoderScorer`). Emit
`--json` so numbers are machine-checked, never hand-entered.

## Dataset / fixtures

Real BEIR under `~/.folklore/bench/<dataset>/<dataset>/` (`corpus.jsonl`,
`queries.jsonl`, `qrels/test.tsv|dev.tsv`), exactly as both bench scripts load it.
Primary `--dataset scifact`; replicate on `nfcorpus`, `fiqa`. For a self-contained
smoke run use `eval/fixtures/scifact-mini/` (40 corpus docs, 18 queries, 19 qrels)
and `eval/scifact-100.json`. Labels are the **qrels** (gold relevant doc ids);
correct-resolve / false-accept are scored against qrels, never the cache gate
(per the honest-metrics discipline in `91-next-experiments.md`). Demand is the
existing Zipfian `drawQ()` (`zW = 1/(i+1+5)^0.9`) over `STEPS` (default 8000).
Paraphrase generalization comes from `para()` adding `SIGMA=0.033` Gaussian noise
(the measured real σ from `bench-paraphrase-sigma.mjs`, `sigma_equiv ≤ 0.15`).

## Variables

**Independent.** Topology ∈ {isolated single-node cache, cooperative federated
pool}, both at the vCache operating threshold whose `false_accept ≤ 0.01`
(`--budget 0.01`). Secondary sweep: peers `P ∈ {4,16,64}`.

**Dependent (new code-level metric).** Net saving per resolved query, three units:

```
saved_i   = ACCEPT_i ? (C_regen − C_retr − C_verify) : (−C_retr)      # miss still pays retrieval
net_total = Σ_i saved_i                                                # over the STEPS stream
net_per_q = net_total / STEPS
fa_cost   = falseAccepts · C_regen      # wrong reuse must be regenerated anyway → no credit
federation_premium = net_per_q(coop) − net_per_q(iso)   # at matched ≤1% false-accept
```

`C_regen`, `C_retr`, `C_verify` are the **measured** tokens/ms from
`probe-unit-costs.mjs` (not the 8000/200 constants). Computed in
`bench-net-savings.mjs` immediately after the operating point is chosen.

## Baseline

The honest reference is the **single-node vCache-tuned cache at matched 1%
false-accept** — `runCurve(false)` operating point — i.e. the number the
literature already reports for semantic caching. Federation's claim is only the
`coop − iso` delta at the same error. (COLD direct-top-1 is reported too, but is
the weak baseline and is *not* the comparison H1 stands on.)

## Procedure

1. `npm run build` (bench scripts import `../dist/...`); `folklore ask "warmup"`
   once; fetch BEIR to `~/.folklore/bench/scifact/scifact/`.
2. Unit costs: `node bench/probe-unit-costs.mjs --json` → `C_regen` (real generate
   call), `C_retr`, `C_verify` (`rerankMatches` over top-K=5), each tokens + p50/p95.
3. Matched-error curves: `node bench/bench-vcache-compare.mjs --dataset scifact --budget 0.01 --json`;
   confirm both arms hit `false_accept ≤ 0.01`, record the two recall@1 operating points.
4. Ledger: `node bench/bench-net-savings.mjs --dataset scifact --budget 0.01 --peers 16 --steps 8000 --json`
   (reads step-2 costs) → `net_per_q` (iso, coop), `federation_premium`, latency
   variant via `bench-memtool-latency.py` p50/p95.
5. Sweep `--peers 4 16 64`, `--dataset nfcorpus fiqa`; cross-check vs latency D3.
6. Verifier ablation: `--verify energy-gate` vs `--verify cross-encoder` to bound
   how much the `C_verify` choice moves the result.

## Statistics & sample size

`n = STEPS = 8000` resolution events (Zipfian), 5 seeds (vary `_s`). Report
`net_per_q` mean ± 95% CI (bootstrap over seeds) and a **paired** test on the
per-seed `coop − iso` delta (Wilcoxon signed-rank, paired because both arms see
the identical stream). **Feasibility note:** RQ4 / E1 flag the *clean-correct base
is small* — SciFact has ~300 qrel queries and far fewer "clean-correct" ones; with
`--budget 0.01` the accepted set shrinks, so per-query CIs widen. Mitigate by
(a) reporting the premium at fixed 1% budget across 3 datasets, (b) pooling seeds,
(c) treating `nfcorpus`/`fiqa` as replications, not just scifact. Power target:
detect a 20% relative premium at α=0.05, power 0.8 — pre-check with the 5-seed
pilot before claiming significance.

## Honest-metrics discipline

Label every number **MEASURED** (probe ms/tokens, in-proc latency) vs
**SIMULATOR** (the Zipfian stream, `para()` noise, CRDT-replicate-to-all deposit —
a boolean peer-holds-tree abstraction, *not* real per-peer retrieval). The
`coop − iso` delta is the only headline; both arms use the **same decision rule
and same 1% budget**, so it is federation coverage, not a weak-baseline artifact.
Folklore-CLI latency is reported separately and never compared in-proc-vs-boot as
equal (the 11 ms core is the production figure). False accepts get **no** token
credit (`fa_cost = falseAccepts·C_regen`).

## The falsifier

Concrete thresholds: H1 is **refuted** if, at `false_accept ≤ 0.01` on scifact,
`federation_premium ≤ 0` tokens/query (overhead erases it), **or** if
`recall(coop) − recall(iso) < 0.02` at the 1% operating point (matched error
collapses the hit-rate gap), **or** if `C_verify ≥ C_regen − C_retr` so every
accepted hit nets ≤ 0 — then the federation premium is a simulator artifact of an
under-tuned isolated baseline and the saving reduces to the single-node number.

## Threats to validity

1. **Boolean deposit abstraction.** `runCurve` replicates the verified
   `{emb, docs:rel}` tree to *all* peers instantly — no real per-peer retrieval or
   CRDT churn/latency. Inflates federated coverage; the E5 pilot, not E4, measures
   real churn. State this; do not promote to a deployment claim.
2. **Unit-cost transfer.** `C_regen`/`C_verify` measured on one model/host may not
   generalize; report the ratio `C_verify / C_regen` so the result is rescalable,
   and run the energy-gate vs cross-encoder verifier ablation.
3. **Small clean-correct base + paraphrase model.** `SIGMA=0.033` is one measured
   point; thin accepted sets at 1% budget widen CIs. Replicate across 3 datasets
   and 5 seeds before any significance claim.
