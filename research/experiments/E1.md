# E1 — Answer-quality-under-reuse in the [0.6, 0.71) similarity band

## Objective & hypothesis

Folklore's headline `hurt = 0` is scored as **retrieval recall@1 over BEIR**
(`bench/bench-inference-tree-sharing.mjs`, `bench/bench-vcache-compare.mjs`). RQ1
warns the falsifier lives *downstream* of retrieval: recall can read ~97% while
end-to-end **answer correctness** silently regresses, because the recalibrated
gate fires at q2q cosine 0.6 — below the paraphrase floor (~0.71) and below every
reliable cache cutoff in the literature (0.78–0.83).

**Hypothesis (falsifiable):** On query pairs whose q2q cosine sits in
`[0.6, 0.71)` *and whose information need genuinely differs* (different gold
qrel), answering via reuse flips a non-trivial fraction of clean-correct answers
to incorrect — `hurt_answer > δ` — even though the same pairs show `hurt ≈ 0` on
recall@1. A per-prompt learned threshold (vCache) holds `hurt_answer ≤ δ` where
`static-0.6` does not.

## What this reuses vs what to build

**Reused, unchanged:**
- `bench/bench-beir.mjs` — real BEIR download + MiniLM (`Xenova/all-MiniLM-L6-v2`,
  384-dim) + sqlite-vec; provides the corpus/query/qrel loader and the
  embedder/index seam (`evalEmbedder`, `openEvalIndex` via
  `scripts/fellows-eval/build-index.mjs`, `lib-corpus.mjs`).
- `scripts/fellows-eval/run-matrix.mjs` + `judge.mjs` + `model.mjs` — the **only**
  end-to-end *answer*-graded harness in the repo: real retrieval
  (`searchGlobal` top-50) → agent model seam (`AKASHIK_EVAL_AGENT_MODEL`, default
  `claude-haiku-4-5`) → `answers.jsonl`; `judge.mjs` grades with Opus and emits
  `judged.jsonl` with a `correct` boolean. SciFact-100 claims are all gold-SUPPORTED.
- `eval/scifact-100.json` — 100 frozen real SciFact claims with `gold_doc_id`.
- `src/domain/query-reuse.ts` — `expandResolvedQueries` / `makeResolvedQueryNode`;
  the read-time hook that swaps a matched resolved-query node for its `answer_docs`
  at the q2q distance — the reuse mechanism under test (gated by
  `FOLKLORE_QUERY_REUSE`).

**To build (named, with slot):**
1. `bench/build-need-pairs.mjs` — pair-set constructor. Embeds every query in a
   dataset (SciFact-100 + BEIR SciFact/NFCorpus/FiQA), computes all pairwise q2q
   cosines, and emits two JSON sets to `eval/fixtures/e1-pairs/<dataset>.json`:
   (a) **HARD near-miss** — pairs with cosine ∈ `[0.6, 0.71)` **and distinct gold
   doc** (different need); (b) **matched paraphrase** control — cosine ∈
   `[0.71, 0.84]` with the *same* gold doc. Each record:
   `{a_qid, b_qid, cos, same_gold}`.
2. `scripts/fellows-eval/reuse-treatments.mjs` — replaces the poison treatments
   (T0/T1/T2) with reuse policies. For query `a`, look up its nearest prior
   resolved-query `b` from the pair set; under policy `P`, if `cos(a,b) ≥ thr(P)`
   the agent is handed **b's verified answer doc as additive context** (per
   `expandResolvedQueries` semantics, RQ1's "reuse-as-context" not blind replace),
   else it answers from fresh top-k. Policies: `fresh`, `static-0.6`, `static-0.8`,
   `vcache`.
3. `bench/vcache-threshold.mjs` — the **per-prompt learned threshold** (vCache,
   arXiv:2502.03771); *absent from the repo* — `bench-vcache-compare.mjs` only
   sweeps one global static `GRID` operating point. Fit logistic regression on
   `(cos, retrieval-correct)` over a held-out calibration split → per-prompt cutoff
   `thr_q` guaranteeing `Pr(correct) ≥ 1−δ`, δ ∈ {1,2,3}%.
4. `bench/score-answer-correctness.mjs` — partitions `judged.jsonl` by q2q bin and
   computes `Δcorrect`, `hurt_answer` (below).

## Dataset / fixtures

Real BEIR via `bench-beir.mjs`: SciFact (5,183 docs / 300 q), NFCorpus
(3,633 / 323), FiQA (8,674 / 1,406), MiniLM + sqlite-vec. Labels = the shipped
`qrels/test.tsv` gold doc; answer-correctness gold = the SUPPORTED verdict for
SciFact-100 claims via `judge.mjs`. **Do not** use `eval/fixtures/scifact-mini` for
headline numbers — its README marks it synthetic and non-comparable; smoke only.
Pair-set sizes are *emergent*: report the actual `[0.6,0.71)` distinct-need count
per dataset (the base is small — see Statistics).

## Variables

- **Independent:** reuse gate policy at fixed encoder —
  `fresh | static-0.6 | static-0.8 | vcache(δ∈{1,2,3%})` (`reuse-treatments.mjs`).
- **Dependent (new code):** end-to-end answer correctness from `judged.jsonl`,
  partitioned by q2q bin. Per bin:
  `Δcorrect = correct(reuse) − correct(fresh)`;
  `hurt_answer = |{q : fresh correct ∧ reuse incorrect}| / |fresh-correct|`.
  Secondary: `hit_rate` (fraction where the gate fired) so the
  hit-rate↔correctness trade is visible. These compute in
  `bench/score-answer-correctness.mjs`; correctness itself is reused from the
  existing Opus judge, *not* re-implemented.

## Baseline

**Answer-fresh** (`fresh` policy: reuse hook off, agent answers from direct top-k)
on the *identical* query stream and identical agent model — the only honest
reference for "did reuse help or hurt the answer." A static-0.8 arm is the
literature-reliable reference; cold recall@1 is **not** an acceptable baseline
here (it answers a different question — retrieval, not answer quality).

## Procedure

1. `node bench/bench-beir.mjs scifact` (then `nfcorpus`, `fiqa`) — download, embed,
   index; confirm the published recall@1/NDCG@10 reproduce.
2. `node bench/build-need-pairs.mjs --datasets scifact,nfcorpus,fiqa` → writes
   `eval/fixtures/e1-pairs/*.json`; print the `[0.6,0.71)` distinct-need count.
3. `node bench/vcache-threshold.mjs --dataset scifact --delta 0.02` → fit per-prompt
   thresholds on the calibration split; emit `eval/out/vcache-thr-<dataset>.json`.
4. Smoke the seam: `AKASHIK_EVAL_BACKEND=stub node scripts/fellows-eval/run-matrix.mjs
   --smoke` to validate wiring with no model spend.
5. For each policy P: `AKASHIK_EVAL_AGENT_MODEL=claude-haiku-4-5
   E1_POLICY=P node scripts/fellows-eval/reuse-treatments.mjs` → appends to
   `eval/out/answers.jsonl` (resumable, keyed by qid|policy).
6. `node scripts/fellows-eval/judge.mjs` (Opus judge, unchanged) → `judged.jsonl`.
7. `node bench/score-answer-correctness.mjs --bins 0.6:0.71,0.71:0.84` →
   `Δcorrect`, `hurt_answer`, `hit_rate` per policy × bin.

## Statistics & sample size

n = number of distinct-need pairs in `[0.6, 0.71)` (emergent; RQ1 flags this base
is *small*). Primary test: **McNemar's exact test** on the paired
fresh-vs-reuse correct/incorrect 2×2 per query (paired by qid) — the right test
for flipped binary outcomes on the same items. Report `hurt_answer` with a
Wilson 95% CI. **Feasibility note:** if a single dataset yields < ~40 clean-correct
distinct-need pairs (underpowered for McNemar at the δ=2% effect), **pool across
SciFact+NFCorpus+FiQA** and report per-dataset plus pooled; if still thin, widen
the agent set (run a second cheaper model) rather than widen the band — the band is
the object of study and must stay `[0.6, 0.71)`.

## Honest-metrics discipline

Label every number **simulator vs measured**: `build-need-pairs`/`vcache-threshold`
geometry is *measured* embedding cosine; answer correctness is *measured* (real
agent + Opus judge), never simulated. Comparisons are **matched** — same query
stream, same agent model, same context size — so only the gate policy varies. No
weak-baseline inflation: the honest delta is `reuse − fresh` and `vcache − static`,
**not** reuse − cold-retrieval. Answers are scored against **gold qrels / gold
SUPPORTED verdict**, never against the cache gate. Retrieval recall@1 (from the two
BEIR benches) is reported *alongside* `hurt_answer` to make the recall↔answer gap
explicit — that gap is the experiment.

## The falsifier

If, in `[0.6, 0.71)`, `static-0.6` yields `hurt_answer > 0.02` (i.e. > δ at the 2%
guarantee) on the pooled distinct-need set **while recall@1-hurt ≈ 0 on the same
pairs**, the 0.6 gate is **falsified** as an answer-quality gate and must rise
toward 0.78–0.83 or go per-prompt. Conversely, if `vcache` holds
`hurt_answer ≤ δ` where `static-0.6` does not, the fix is the *learned threshold*,
not the constant. Null/non-refutation: `Δcorrect ≥ 0` and `hurt_answer ≤ δ` for
`static-0.6` across all three datasets vindicates the shipped gate.

## Threats to validity

1. **Thin distinct-need base.** `[0.6, 0.71)` ∩ different-gold pairs may be rare on
   small BEIR sets, underpowering McNemar — mitigated by pooling datasets/agents,
   and by *reporting the count* rather than padding with paraphrase pairs.
2. **Judge–agent confound.** Opus judges Haiku answers; an over-lenient judge masks
   `hurt`. Mitigate with a held-out human-spot-check on a 20-pair sample and an
   EM/BERTScore cross-check where the answer is extractive.
3. **Encoder-specific cosine.** The `[0.6, 0.71)` band is MiniLM-specific; cutoffs
   are non-portable across encoders (RQ1). Findings bind to MiniLM — re-run
   `build-need-pairs` per encoder before generalizing the numeric thresholds.
