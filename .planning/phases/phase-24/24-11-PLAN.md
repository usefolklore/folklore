    - `akashik migrate v5 --rollback` → restore from backup

    Migration flow (per research lines 510-545):

    1. **Detect.** Read first 10 nodes from ~/.akashik/graph.json (raw JSON.parse, NOT fromJson — the v5 type lacks `room`). If NONE has a `room` field AND no rooms.json AND no shared-rooms.json: print "Already on V5." and exit 0.

    2. **Backup.** Atomic copy ~/.akashik/graph.json → ~/.akashik/graph.v4-backup.json. If backup already exists (previous run crashed mid-migration), prompt to confirm overwrite.

    3. **Load v4 data.** Read graph.json as raw JSON. Walk nodes — for each: extract optional `room` (string), copy all other fields.

    4. **Heuristic workspace inference (per research lines 522-528):** For each unique room name:
       - Slugify it (lowercase, dashes).
       - Check if a directory at `~/personal/<slug>` or `~/code/<slug>` exists. (Configurable repo-roots list from config.yaml — read but with sane defaults.)
       - If match: workspace = slug. Else: workspace = undefined.

    5. **Transform nodes:** drop `room` field, set `private: false` (default), set `workspace: <inferred>` where heuristic matched. Write graph.json atomically (.tmp + rename — match Phase 20 pattern from sessions-state.ts).

    6. **Flatten reputation.** Read peer-reputation.json (best-effort; may not exist). For each peer.subjects: drop entries with key matching `/^room:/`. Write back atomically. Surface a count: "Flattened N entries; M peers lost reputation entirely."

    7. **Delete v4 artifacts:** rm ~/.akashik/rooms.json, rm ~/.akashik/shared-rooms.json, rm ~/.akashik/shared-rooms.json.lock (if present). Leave per-room .ydoc files in place for now (a v5 boot will create graph.ydoc; the old .ydoc files are orphaned but harmless — could be cleaned in Phase 25+).

    8. **Verify + summary.** Print exactly the format from CONTEXT.md lines 142-161:
       ```
       Reading ~/.akashik/graph.json...
         21,128 nodes found across 5 rooms + 6 system rooms.
       ...
       Migrating to V5...
         ✓ Stripped `room` field from 21,128 nodes
         ✓ Set `private: false` on 21,128 nodes
         ✓ Heuristic workspace assignment: 3,481 nodes tagged
         ✓ Flattened 4 reputation entries
         ...
       V5 cutover complete. Run `akashik doctor` to verify.
       ```

    Rollback (`--rollback`):
    9. Read graph.v4-backup.json. Atomic copy back to graph.json.
    10. Print warnings (per CONTEXT.md lines 551-559): rooms.json and shared-rooms.json deletions are not auto-recoverable; reputation flattening is not auto-reversible. Tell the user they can re-run migrate v5 or restore rooms.json from their own backup.
