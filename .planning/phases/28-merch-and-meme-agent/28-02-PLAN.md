<objective>
Scaffold a standalone autonomous Twitter meme-agent under `src/agents/meme-agent/`
that runs the full generate → post → append pipeline, DRY-RUN by default, posting
nothing live until X credentials exist. It defines the `site/assets/memes.json`
schema (consumed by Plan 28-03), generates a no-credit templated SVG meme from
existing folk art as the default, reuses the project's existing X client for the
(gated) post step, and appends a record to memes.json.

Purpose: AGENT-01 — "memes by the network, for the network", scaffolded so it is
runnable the moment X creds land, with zero live posting / account creation / spend
in this milestone. It also produces the data contract Plan 28-03 wires into the site.

Output: 4 TS files under src/agents/meme-agent/ (compiling under the existing tsc),
a seeded site/assets/memes.json, and docs/agents/meme-agent.md.

LANDING DIRECTORY DECISION: the agent lives under `src/agents/meme-agent/` (NOT a
top-level tools/ dir) because tsconfig.json has `rootDir:"./src"` +
`include:["src/**/*"]` and `lint:"eslint src tests"` — code outside src/ would not
compile under `npm run build` nor be linted. This is the only way to honor the
"tsc stays 0" + "compiles" constraint. Note this rationale in the SUMMARY.
</objective>
